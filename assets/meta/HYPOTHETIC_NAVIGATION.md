# Hypothetic Navigation — Infra Pitch Research App

## 1. Главный экран (Home)
- **Хиро-блок «Infra Pitch Research»**: кратко объясняет пакет (4 DR-исследования).
- **Быстрые плитки**: DR1, DR2, DR2.5, DR3 с основными KPI (MTTR −40%, 20+ офферов, Trust Center ≤5 дней, ROI 17.6×).
- **CTA-кнопки**:
  - `Скачать ZIP` (ведёт на пакет с .md/.docx/.pdf).
  - `Поделиться ссылкой` (system share sheet).
- **Статус версии**: дата сборки, номер релиза.

## 2. Каталог исследований (Research Library)
- Таб в нижнем навбаре.
- Список карт «DRX — Название» с подзаголовками:
  - `Жанр` (Architecture / Portfolio / Compliance / Value).
  - `Объём` (кол-во слов/экранов).
  - `Обновлено` (дата).
- Фильтры/теги (checkbox chips): Architecture, Roadmap, Compliance, Financials.
- Поиск по названию/ключевым словам (быстрый индекс по заголовкам Markdown).

## 3. Экран исследования (Detail View)
- Sticky header с названием, кнопками `PDF`, `DOCX`, `Поделиться`.
- Разделы прокручиваются по якорям (используем `flutter_markdown` + `ScrollablePositionedList`).
- Слева (web/tablet) или в нижнем листе (mobile) — оглавление:
  1. Резюме
  2. Контекст и цель
  3. Ключевые выводы
  4. Детализация (подразделы)
  5. Риски и ограничения
  6. Финальные рекомендации
  7. Источники
- Плашки «Сервисы»:
  - `Скопировать цитату` (копирует выделенный текст).
  - `Отправить в Telegram` (deeplink на @plane_logger_bot с шаблоном).

## 4. Источники (Sources Hub)
- Поддерживает вывод объединённого списка источников по всем DR.
- Фильтрация по типу: `internal (eng/notes/*, rus/notes/*)`, `external (articles, case studies)`.
- Тап раскрывает модалку со ссылкой/описанием.
- Кнопка `Экспорт CSV` (генерирует временный файл и шарит).

## 5. Таймлайн обновлений (Changelog)
- Экран с хронологией изменений (по примеру research_log.md).
- Каждая карточка: дата, что обновили, какие файлы.
- Кнопка `Синхронизировать` (будущее место для API обновления).

## 6. Нижний навбар
1. Home
2. Library
3. Sources
4. Timeline
5. Profile (плейсхолдер для клиента: контакты дуэта, SLA, secure upload).

## 7. Профиль/Контакты
- Информация о дуэте (роль, часовые пояса, SLA).
- Кнопка `Запросить апдейт` (открывает форму).
- Ссылки на Telegram/Email.

## 8. Оффлайн/кеш
- Assets содержат `.md` + предсгенерированные `.json` для оглавлений.
- При первом запуске — быстрая индексация (isolate) и сохранение в локальный cache для поиска.

## 9. Что получит заказчик
- Кроссплатформенное приложение с офлайн-доступом.
- Единую точку входа ко всем DR с PDF/DOCX скачиваниями.
- Навигацию по разделам, источникам и истории версий.
