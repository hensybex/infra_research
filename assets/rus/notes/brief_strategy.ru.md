# Исследование 2 Solution Pathing — бриф и стратегия

## Цель
Сформировать портфель решений и дорожную карту для российского инфраструктурного провайдера (уровня Timeweb/Selectel), чтобы обосновать работу компактной AI-команды из инженера и продуктового стратегa.

## Артефакты
- `assets/eng/reports/DR2_solution_pathing.md` (и локализация `assets/rus/reports/DR2_solution_pathing.ru.md`) — клиентский нарратив: обзор, discovery-каталог, roadmap, staffing, риски, приложения.
- `hensybex/projects/infra_pitch/reports/DR2_solution_pathing.md` — итоговый экспорт за пределы рабочей директории.
- `assets/eng/notes/DR2_decision_log.md` + `assets/rus/notes/DR2_decision_log.ru.md` — обоснование выборов.
- Зеркала источников в `sources/DR2/`.
- Актуальный `research_log.md` и `inventory.json`.

## Критерии успеха
- 20–30 прикладных AI/agent-концептов, адаптированных под российского провайдера.
- Флагманская roadmap с Console Copilot, AI marketplace и сервисными модулями (6–12 месяцев, метрики, зависимости, быстрые победы).
- Чёткое объяснение преимущества компактной команды (multi-orchestrator, MCP, vector, mobile/desktop, DevOps-автоматизация, compliance-aware).
- Интеграция глобальных референсов (AWS Agent Factory, Azure AI Foundry, GitHub Agent HQ) и контекста РФ/СНГ.
- Риски и допущения для финансов в DR#3.
- Учет комплаенса/резидентности.

## Ограничения и процесс
- Лимит: 90 минут, не более 16 поисковых запросов / 20 API-запросов.
- Чекин каждые 15 минут (лог + при необходимости Telegram).
- Соблюдать SOP глубоких исследований: макро-скан, карта гипотез, мультиветвевое исследование, консолидация, финальный отчёт.

## Стратегия
1. Сканировать макрофон: модели marketplace, консольные ассистенты, инициативы RU-инфраструктуры, требования комплаенса.
2. Выявить паттерны решений, совпадающих с компетенциями двух ключевых ролей (MCP, multi-orchestrator, vector workflows, mobile, Dagger-автоматизация).
3. Приоритизировать 20–30 решений по потокам: внутренние операции, клиентский marketplace, партнёрская экосистема.
4. Сформировать фазы roadmap (Foundation → Pilot → Expansion), указать зависимости, метрики, ресурсы.
5. Оценить риски, допущения, необязательные ветки.
6. Задокументировать ход мыслей, обновить артефакты, передать итоговый отчёт.
