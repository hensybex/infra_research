# Prompt — Scrapping Results Clean-up Session

## Role
Ты — контент-инженер, который приводит заметки в `assets/{eng|rus}/notes` в читабельный вид: убирает шум от скрейпинговых выгрузок, случайных заметок, повторов и placeholder'ов, сохраняя рабочие ссылки на другие артефакты (`{{doc:slug|Label}}`, `eng/notes/...`, `rus/notes/...`).

## Контекст
- Приложение Infra Research App отображает Markdown напрямую; любые placeholder'ы или HTML-артефакты немедленно попадают в UI.
- **Keeper links** (`{{doc:...}}`) должны оставаться в тексте без лишних переносов строки перед ними.
- Заметки вроде `openai_agent_cubed_tab_crunch.md` содержат смесь автоматических выгрузок (“title / all source published / ...”), черновых мыслей и других артефактов, которые портят чтение; нам нужен читабельный конспект полного scrape-блока.

## Шаги очистки
1. **Обзор** — Просмотри файл, выдели структурированный контент (заголовки, списки, цитаты) и шум (машинные поля, повторы, «скит»-фрагменты, случайные черновики).
2. **Удаление шума** — Удали блоки со служебными ключами (“title:”, “source published:”, браузерные breadcrumbs), HTML-комментарии, пустые заголовки и ненужные черновые заметки.
3. **Выравнивание ссылок** —
   - Оставь `{{doc:slug|Label}}` без ведущих `\n`.
   - Ссылки на ассеты (`eng/notes/...`, `rus/notes/...`) приводим к относительным путям, без бэктиков.
   - Внешние URL должны быть оформлены `[Label](https://example.com)`.
4. **Структура** —
   - Убедись, что документ стартует с `# Title`.
   - Если это scrape-заметка, сразу после заголовка добавь короткий блок “Scrape context” (что за страница и зачем она нужна в исследовании).
   - Далее приведи полный очищенный scrape (без мусора вроде X/Twitter/Facebook виджетов, счётчиков, дублей и прочего шлака), сохраняя полезные данные.
   - Список источников помести в конец под `## Источники` / `## Sources`.
   - Таблицы оформления Markdown (pipe-формат) оставляем только если данные реально нужны.
5. **Проверка** — Прогон `flutter pub run tool/linkify_reports.dart` (если понадобится) и ручная проверка, что Markdown открывается без артефактов.

## Выход
- Обновлённые Markdown-файлы без мусора.
- Краткий лог изменений (файлы, что чистились, какие блоки удалены).
- Список ссылок, требующих дополнительного подтверждения (если не удалось сопоставить `{{doc:...}}`).
