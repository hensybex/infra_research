# CONTINUE

## Контекст текущей сессии
- Строим единый реестр документов (Hive) и хотим, чтобы все ссылки внутри отчётов были кликабельными, без упоминаний `.md`.
- Все отчёты/заметки конвертированы в формат `{{doc:slug|label}}`; для разрешения используется `RegistryMarkdown` + `SourceLauncher`.
- Источники DR1/DR2 скопированы в `assets/{eng,rus}/sources`, добавлены в реестр; `DocumentRepository` умеет читать их напрямую с диска, даже если они ещё не попали в asset bundle.
- `_documentSeeds` теперь содержит явный список всех отчётов/заметок/источников; удалили незавершённый вызов `_noteSeeds(...)`, из‑за которого падала сборка.
- Чтобы nested `sources` попали в bundle, `pubspec.yaml` теперь перечисляет подкаталоги `assets/{eng,rus}/sources/dr1` и `dr2`.
- Note-preview заменён на полноэкранную навигацию: `SourceLauncher` роутит на `NoteDetailScreen` через `GoRouter`, а для отчётов продолжает пушить `/report/:id`.

## Что осталось проверить/сделать
1. Собрать приложение заново (`flutter run -d macos`), чтобы убедиться, что новые источники попали в asset bundle и переход по ссылкам ведёт на нужные экраны (report/detail).
2. Если SourceLauncher всё ещё показывает алерт, снять лог `[RegistryMarkdown] tap slug=...]` + `Asset load failed ...` — это укажет на конкретный путь.
3. После успешного теста можно отключить `_resetOnLaunch` в `DocumentRegistry`, когда контент стабилизируется.
4. В перспективе убрать disk-fallback и оставить только bundle, когда все ассеты гарантированно включены.
